# PHASE DOCUMENT — Template Generation Persistence

## Section 1 — Feature

Extends POST /generation/from-template to persist template-generated output to
documentsTable after generation completes, making template-generated documents
promotable to the registry. The route handler serialises the template output,
runs the existing AI-based 19-check QC pipeline, inserts a document record, and
returns an extended response including a document ID. The From Template result
panel in generate.tsx gains a "View in Registry" link and a conditional
"Promote to Registry" button.

---

## Section 2 — Why this is necessary

Template generation currently produces governed, compliance-checked output that
is immediately discarded — nothing is written to documentsTable, no document ID
is returned, and the user cannot promote template-generated content to the
registry. The compliance value of the Template Registry (required ACU enforcement,
prohibited ACU checking, section-level QC) is wasted if the output cannot be
saved. Every template-generated document must currently be recreated via custom
generation to enter the registry, defeating the purpose of the template path.

---

## Section 3 — Current state

- artifacts/api-server/src/routes/generation/index.ts: implements POST
  /generation/from-template which calls generateFromTemplate() and returns
  its result directly with no persistence. Also implements POST
  /generation/generate (custom path) which persists to documentsTable
  immediately after generation and runs runQC(). The runQC(),
  getLockedACUs(), buildACUPromptSection(), and getComplianceConstants()
  functions are all defined and used in this file.
- artifacts/api-server/src/lib/generationEngine.ts: exports
  generateFromTemplate(request) which returns { template_id, output:
  Record<string, string>, compliance_check: { pass, issues[] }, metadata }.
  Must not be modified.
- lib/db/src/schema/documents.ts: defines documentsTable with all required
  columns. No schema changes needed.
- artifacts/unlock-intel/src/pages/generate.tsx: From Template result panel
  currently shows compliance badge, issues, named sections, and metadata. No
  promote button and no registry link exist because no document ID is returned.

### 5.2 — Extend POST /generation/from-template route handler

File: artifacts/api-server/src/routes/generation/index.ts

After the call to generateFromTemplate(request) returns, add the following
steps in sequence before the response is sent. Do not modify any code before or
after the existing call to generateFromTemplate() except to add these steps.

Step A — Serialise output to markdown

Convert result.output (type Record<string, string>) to a single markdown
string using the following rules:

- Section order: use the order in which keys appear in result.output as
  returned — do not sort or reorder
- For each section: render as ## [section_label]\n\n[content]
- Section label: use the key name from result.output as the heading (e.g.
  if the key is "introduction", render ## introduction)
- Empty sections: if a section's value is an empty string or contains only
  whitespace, omit it entirely — do not render a heading with no content
- Join sections with \n\n between them
- Result: a single serialisedContent string

Step B — Run AI QC

Call runQC(serialisedContent, complianceText, templateName, 1) where:
- serialisedContent is the string from Step A
- complianceText is obtained by calling getComplianceConstants() then
  formatting it the same way the custom generation path does
- templateName is result.metadata.template_name (used as documentType parameter)
- attempt is 1

Store the result as aiQcReport.
Set reviewState to "CLEAN" if aiQcReport.overall === "pass",
otherwise "REQUIRES_REVIEW".

Step C — Insert document record

Insert a new row into documentsTable. For all fields not explicitly listed
below, mirror the insert pattern from POST /generation/generate in the same
file exactly — use the same field names, defaults, and patterns. Explicitly set:

- id: nanoid() (or the local nanoid pattern confirmed in 5.1)
- file_code: "gen_" + nanoid(8)
- name: request.context?.document_name ?? "Template: " + result.metadata.template_name
- type: "generated"
- tier: 3
- category: "Generated"
- lifecycle_status: "DRAFT"
- review_state: reviewState (from Step B)
- is_generated: true
- content: serialisedContent (from Step A)
- output_type: result.metadata.template_name ?? "whitepaper"
- qc_report_id: aiQcReport.overall (stores "pass" or "fail" — follow existing pattern)
- qc_history: [aiQcReport]
- version: 1
- last_reviewed: new Date().toISOString()
- description: request.context?.document_name ?? result.metadata.template_name

Store the inserted document's id as documentId.

Step D — Insert changelog entry

Insert a changelog entry mirroring the pattern used in POST /generation/generate
in the same file exactly — same fields, same structure. Set:
- action: "DOCUMENT_GENERATED"
- document_id: documentId
- details: "Generated from template: " + result.metadata.template_name
- triggered_by: "template_generation"

Step E — Return extended response

Return the original generateFromTemplate() response shape with three additional
fields appended. Do not rename, remove, or restructure any existing fields:

{
  "template_id": "...",
  "output": { ... },
  "compliance_check": { ... },
  "metadata": { ... },
  "document_id": "[documentId]",
  "review_state": "[reviewState]",
  "ai_qc_report": { ... }
}

compliance_check in the response reflects the rule-based result from
generateFromTemplate() only. review_state reflects the AI QC result only.
These must not be swapped, combined, or cross-referenced.

## Section 6 — Acceptance criteria

AC1. POST /generation/from-template with a valid template_id returns a response
containing all original fields (template_id, output, compliance_check, metadata)
plus three new fields: document_id (non-empty string), review_state ("CLEAN" or
"REQUIRES_REVIEW"), and ai_qc_report (object with overall, checks, fail_count fields).

AC2. After calling POST /generation/from-template, a new row exists in
documentsTable with lifecycle_status: "DRAFT", is_generated: true, tier: 3,
category: "Generated", and content containing the serialised markdown output.
Manual test required: query DB — SELECT id, lifecycle_status, tier, category,
is_generated, length(content) FROM documents WHERE id = '[document_id]'.

AC3. The review_state field in the response and in documentsTable is "CLEAN"
when ai_qc_report.overall === "pass" and "REQUIRES_REVIEW" when
ai_qc_report.overall === "fail". It is not derived from compliance_check.

AC4. The serialised content in documentsTable contains one ## [key] heading
per non-empty section from output, in the order the keys appear in the response,
with sections joined by \n\n. Empty or whitespace-only sections are omitted.
Manual test required: inspect content column for a generated document.

AC5. A changelog entry exists for the generated document with action:
"DOCUMENT_GENERATED" and document_id matching the returned document_id.
Manual test required: SELECT * FROM changelog WHERE document_id = '[id]'.

AC6. The From Template result panel in /generate shows a "View in Registry"
link when generation succeeds. Clicking it navigates to /registry/[document_id].
Manual test required: visual inspection in browser.

AC7. The "Promote to Registry" button is visible and enabled when
review_state !== "REQUIRES_REVIEW". Clicking it calls POST
/generation/[document_id]/promote and on success shows "Document promoted to
registry." Manual test required: visual inspection and network tab inspection.

AC8. When review_state === "REQUIRES_REVIEW", the promote button is visible
but disabled with label "Promote to Registry" and note "Fix compliance issues
before promoting." Manual test required: visual inspection using a template that
produces an AI QC failure, or by directly querying the DB for a document with
review_state "REQUIRES_REVIEW" and verifying the button state.

AC9. POST /generation/generate (custom mode) behaviour is unchanged — same
request, same response shape, same persistence pattern. Manual test required:
run a custom generation and verify it completes without error.

AC10. generateFromTemplate() in artifacts/api-server/src/lib/generationEngine.ts
is unmodified. Manual test required: confirm git diff contains no changes to this file.

AC11. No changes to lib/db/src/schema/documents.ts. Manual test required:
confirm git diff contains no changes to this file.
