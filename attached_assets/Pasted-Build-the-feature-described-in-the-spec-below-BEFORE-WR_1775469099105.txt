## Section 1 — Feature

Adds the database foundation for the investor intelligence and belief state system. Four new tables are created (belief_registry, lead_beliefs, belief_transitions, lead_intelligence), the documents table gains a `belief_targets` column, and the leads table `notes` field is migrated from jsonb to a plain text field. The 22 beliefs are seeded into the registry on server startup.

---

## Section 2 — Why this is necessary

Without this schema, no belief state can be tracked per lead, no intelligence record can be persisted, and no document can be mapped to the beliefs it targets. All subsequent phases (010b–010d) depend on these tables existing and the 22 beliefs being seeded. The current `notes` field on leads is JSONB but unused — converting it to text now avoids a more complex migration later when the frontend starts writing to it.
