## Section 1 — Feature

Adds four LLM-backed endpoints to the belief state system and injects belief context into the existing transcript analysis flow. The new endpoints are: `POST /leads/:leadId/beliefs/analyze` (detect belief signals from transcript), `GET /leads/:leadId/beliefs/next` (next belief gap + recommended document), `GET /leads/:leadId/beliefs/gates` (gate status), and `POST /leads/:leadId/intelligence/generate` (generate full intelligence profile from notes + transcript). The existing `POST /recommendation/analyze` endpoint is surgically extended to inject belief states and intelligence summary into the Claude prompt when a `lead_id` is provided.

---

## Section 2 — Why this is necessary

The belief tables and CRUD endpoints from 010a/010b1 are scaffolding only. Without the LLM analysis endpoint, belief states must be set manually. Without the gates endpoint, the recommendation engine cannot enforce hard gates. Without `intelligence/generate`, the profile intelligence feature has no intelligence layer. Without the analyze context injection, the recommendation engine ignores everything learned about the investor.

### 5.1 — New endpoints in lead-beliefs/index.ts

File: `artifacts/api-server/src/routes/lead-beliefs/index.ts` (additive — three new endpoints)

All three new routes must be declared BEFORE `PATCH /leads/:leadId/beliefs/:beliefId`.

**GET /leads/:leadId/beliefs/next**

Logic:
1. Fetch all 23 beliefs from `beliefRegistryTable`
2. Fetch all `lead_beliefs` rows for this lead
3. Build state map: belief_id → current state (default `'UNKNOWN'` if no row)
4. Filter to `policy_status = 'active'` beliefs only
5. Skip beliefs where a `lead_beliefs` row exists with `investor_relevance = 'not_applicable'`
6. Find highest-priority gap: state is `'UNKNOWN'` or `'ABSENT'`
7. Priority order: universal (U1→U4), then lead's cluster beliefs (from `leadIntelligenceTable.cluster` if set), then company_conviction, then founding_round
8. Return: `{ next_belief: BeliefRegistry | null, current_state: string | null, recommended_document_id: string | null }`
   - `next_belief`: full belief registry row for highest-priority gap, or null if all active beliefs are PARTIAL/ESTABLISHED/BLOCKED
   - `recommended_document_id`: `primary_document_id` from belief registry (direct lookup only)

Error: 500 `{ error: 'Failed to fetch next belief' }`

---

**POST /leads/:leadId/beliefs/analyze**

Logic:
1. Body: `{ transcript: string }` — required. If missing or empty: 400 `{ error: 'transcript is required' }`
2. Fetch all beliefs from `beliefRegistryTable` where `policy_status = 'active'`
3. Fetch `leadIntelligenceTable` for this lead to determine cluster
4. Select relevant beliefs: all universal + cluster beliefs (if known) + company_conviction + founding_round. If cluster unknown, include all active.
5. Call Claude:

```
You are analysing a sales call transcript to detect investor belief signals.

For each belief listed below, determine from the transcript whether the investor showed evidence of holding it, actively rejected it, or showed no signal.

BELIEFS TO ANALYSE:
[For each belief: "ID: [id] | Name: [name] | Description: [description if present, else name]"]

TRANSCRIPT:
[transcript text]

Return a JSON array only — no other text. Each element:
{
  "belief_id": "[ID]",
  "signal": "ESTABLISHED | PARTIAL | ABSENT | BLOCKED | UNKNOWN",
  "evidence_quote": "[exact quote max 2 sentences — omit field if no direct quote]",
  "confidence": "HIGH | MEDIUM | LOW"
}

Rules:
- Do not infer. If no signal in the transcript: return UNKNOWN.
- A question about a topic does NOT establish the belief — it suggests ABSENT or PARTIAL.
- Engagement with a topic is PARTIAL. Explicit statement of understanding is ESTABLISHED.
- An objection that cannot change with information is BLOCKED (dispositional).
- An objection that could change with the right information is ABSENT (correctable).
- Return exactly one object per belief in the input list.
```

Model: `claude-sonnet-4-6`, max_tokens: 8192

6. Parse: regex extract JSON array, then JSON.parse. If fails: 500 `{ error: 'Failed to parse Claude response' }`

7. For each result:
   - If `confidence = 'HIGH'` or `'MEDIUM'` AND `signal !== 'UNKNOWN'`: upsert `lead_beliefs` row (same logic as PATCH — write transition if state changes), set `evidence_source: 'call_transcript'`, `confidence`, `evidence: evidence_quote`, `updated_by: 'claude_analysis'`
   - If `confidence = 'LOW'`: do NOT update DB — include in response as preview only

8. Return:
```json
{
  "results": [
    {
      "belief_id": "U1",
      "belief_name": "The Practical Advice Gap",
      "signal": "PARTIAL",
      "confidence": "HIGH",
      "evidence_quote": "...",
      "updated": true
    }
  ],
  "updated_count": 3,
  "preview_count": 2
}
```

Error: 500 `{ error: 'Failed to analyze beliefs' }`

---

**GET /leads/:leadId/beliefs/gates**

Logic:
1. Fetch `lead_beliefs` rows for this lead
2. Build state map: belief_id → state
3. Evaluate:
   - `U4_established = state_map['U4'] === 'ESTABLISHED'`
   - `F0_established = state_map['F0'] === 'ESTABLISHED'`
   - `F1_established = state_map['F1'] === 'ESTABLISHED'`
   - `F2_established = state_map['F2'] === 'ESTABLISHED'`

4. Return:
```json
{
  "gates": {
    "can_ask_risk_appetite_question": { "open": true/false, "reason": "U4 established" / "U4 not yet established" },
    "can_recommend_pack_1": { "open": true/false, "reason": "F0 established" / "F0 not yet established" },
    "can_recommend_pack_2": { "open": true/false, "reason": "F0 established" / "F0 not yet established" },
    "can_make_investment_ask": { "open": true/false, "reason": "..." }
  },
  "belief_states": {
    "U4": "[state or UNKNOWN]",
    "F0": "[state or UNKNOWN]",
    "F1": "[state or UNKNOWN]",
    "F2": "[state or UNKNOWN]"
  }
}
```

`can_make_investment_ask` requires F0 AND F1 AND F2 all ESTABLISHED.
Reason for `can_make_investment_ask`: "F0, F1, F2 established" if open, "F0 not yet established" if F0 not established, "F0 established but F1 or F2 not yet established" if F0 established but F1 or F2 not.

Error: 500 `{ error: 'Failed to fetch gate status' }`

  "hot_button": "family_security | freedom | legacy | relief | significance | null",
  "hot_button_confirmed": false,
  "hot_button_quote": "exact quote from transcript if detected | null",
  "spin_situation": "their current situation in one sentence | null",
  "spin_problem": "the core problem they face | null",

  "spin_need_payoff": "what success looks like for them | null",

  "blocker_type": "dispositional | correctable_misconception | null",

const intelligenceRows = await db.select().from(leadIntelligenceTable)
