valid (the DB won't catch these), but internally coherent.

A data integrity failure is when two fields on the same record contradict
each other, a record's state is logically impossible, or related records
have drifted out of sync.

# SECTION 3 — PROMPT INTEGRITY (058)

Assess whether LLM prompts are still fit for purpose — using current data,
producing correct output schemas, and genuinely context-sensitive.

Only run this section if the project has LLM PROMPTS listed in the
project context. If none: write "NOT APPLICABLE — no LLM prompts."

A prompt integrity failure is when:
- A prompt was written before new data was added and doesn't use it
- A prompt's output schema no longer matches what the code expects
- A prompt produces the same output regardless of its inputs (hollow)
- A prompt contains hardcoded values that should come from the DB
