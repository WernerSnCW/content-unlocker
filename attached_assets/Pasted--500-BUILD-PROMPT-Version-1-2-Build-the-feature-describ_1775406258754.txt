# PHASE DOCUMENT — Template Generation Content Format Fix

Version: v1.0

## Section 1 — Feature

Adds a plain-text format constraint to the prompt in generateFromTemplate()
so that Claude returns section content as readable prose rather than structured
JSON objects. One sentence addition to one prompt instruction in one file.

---

## Section 2 — Why this is necessary

The template generation prompt tells Claude to return a JSON object with section
IDs as keys but never specifies what format each value should be. Claude
sometimes returns structured JSON objects as section values. The string coercion
in generateFromTemplate() then converts these to escaped JSON strings via
JSON.stringify(), producing unreadable content like {"title":"Qualification
Gate","required":true,...} in the Document Registry and in persisted documents.

---

## Section 3 — Current state

- artifacts/api-server/src/lib/generationEngine.ts: generateFromTemplate()
  builds a prompt from multiple parts joined by "\n". The final part appended
  to promptParts is: "Return a JSON object with each section ID as a key and
  the generated content as the value. Include a _metadata key with template_id,
  word_counts, acus_used, and compliance_check status."
- This instruction does not specify the format of each section value.
- The string coercion at lines 221-228 converts non-string values via
  JSON.stringify(). This must remain unchanged.

## Section 5 — Required behaviour

### 5.1 — Modify the final prompt instruction in generateFromTemplate()

File: artifacts/api-server/src/lib/generationEngine.ts
Function: generateFromTemplate()

Find the line that appends the final instruction to promptParts. It currently
reads (or is functionally equivalent to):

"Return a JSON object with each section ID as a key and the generated content
as the value. Include a _metadata key with template_id, word_counts, acus_used,
and compliance_check status."

Replace it with:

"Return a JSON object with each section ID as a key and the generated content
as the value. Each section value must be a plain text or markdown prose string —
never a JSON object, array, or structured data type. Include a _metadata key
with template_id, word_counts, acus_used, and compliance_check status."

The only change is the insertion of the sentence:
"Each section value must be a plain text or markdown prose string — never a
JSON object, array, or structured data type."

Do not change anything else in this function. Do not change the string coercion
logic. Do not change the parsing logic. Do not change any other file.
