## Section 1 — Feature

Adds the background analysis engine for the Work Queue. One new endpoint: `POST /work-queue/start`. When called, it creates a session, fetches all open Review tasks, looks up their linked documents, fetches compliance constants from the DB, and calls Claude to analyse each document. Findings are classified as `auto_fix` (objective compliance violations) or `decision_card` (content quality issues) and persisted to `work_queue_findings`. Session progress is updated after each document. Only one active session at a time — a second POST /start while ANALYSING or PENDING returns the existing session.

---

## Section 2 — Why this is necessary

Without the analysis engine, the work queue session model has no data. The frontend polling endpoint returns PENDING forever. This phase is what populates findings so Phase 011c can serve them.

### 5.1 — POST /work-queue/start

File: `artifacts/api-server/src/routes/work-queue/index.ts` (additive — new endpoint)

Declare BEFORE any future parameterised routes. This is a static route.

**Logic:**

**Step 1 — Concurrency guard**
Check for an existing active session (status IN ('PENDING', 'ANALYSING', 'READY')):
```typescript
const existing = await db.select().from(workQueueSessionsTable)
  .where(inArray(workQueueSessionsTable.status, ['PENDING', 'ANALYSING', 'READY']))
  .limit(1);

if (existing[0]) {
  return res.status(200).json({ session: existing[0], already_running: true });
}
```

**Step 2 — Fetch open Review tasks with linked documents**
```typescript
const openTasks = await db.select().from(tasksTable)
  .where(and(
    eq(tasksTable.status, 'Open'),
    eq(tasksTable.type, 'Review'),
    isNotNull(tasksTable.linked_document_id)
  ));
```

If no open Review tasks with linked documents: create a session with status COMPLETE, total_tasks: 0, return it.

**Step 3 — Fetch documents for those tasks**
```typescript
const documentIds = [...new Set(openTasks.map(t => t.linked_document_id as string))];
const documents = await db.select().from(documentsTable)
  .where(inArray(documentsTable.id, documentIds));
```

Filter to documents where `lifecycle_status = 'CURRENT'` and `review_state = 'REQUIRES_REVIEW'`. If a task's linked document is not found or doesn't meet these criteria, skip the task (do not create a finding for it).

**Step 4 — Fetch compliance constants**
```typescript
const constants = await db.select().from(complianceConstantsTable)
  .where(eq(complianceConstantsTable.status, 'ACTIVE'));
```

Build two lookup structures:
```typescript
// Prohibited values — things that must NEVER appear
const prohibitedValues = constants.filter(c => c.is_prohibited);

// Canonical replacements — correct values keyed by key
const canonicalValues: Record<string, string> = {};
constants.forEach(c => {
  if (!c.is_prohibited) canonicalValues[c.key] = c.value;
});
```

**Step 5 — Create session**
```typescript
const session = await db.insert(workQueueSessionsTable).values({
  status: 'ANALYSING',
  total_tasks: eligibleDocs.length,
  analysed_tasks: 0,
}).returning();
const sessionId = session[0].id;
```

Return the session immediately (202 Accepted):
```typescript
res.status(202).json({ session: session[0], already_running: false });
```

**Step 6 — Run analysis in background (after response is sent)**

Use `setImmediate` or `process.nextTick` to run analysis after the response:

```typescript
setImmediate(async () => {
  try {
    await runAnalysis(sessionId, eligibleDocs, eligibleTasks, prohibitedValues, canonicalValues);
  } catch (err) {
    await db.update(workQueueSessionsTable)
      .set({ status: 'FAILED', error_message: String(err), completed_at: new Date() })
      .where(eq(workQueueSessionsTable.id, sessionId));
  }
});
```

### 5.2 — runAnalysis function

Define as a standalone async function in the same file (not inside the route handler):

```typescript
async function runAnalysis(
  sessionId: string,
  documents: Array<typeof documentsTable.$inferSelect>,
  tasks: Array<typeof tasksTable.$inferSelect>,
  prohibitedValues: typeof complianceConstantsTable.$inferSelect[],
  canonicalValues: Record<string, string>
): Promise<void>
```

For each document (processed sequentially — not in parallel, to avoid rate limits):

**5.2a — Build compliance context string for Claude**

```typescript
const prohibitedList = prohibitedValues
  .map(p => `- PROHIBITED: "${p.value}" (${p.label}) — ${p.prohibited_reason || 'must not appear'}`)
  .join('\n');

const canonicalList = Object.entries(canonicalValues)
  .map(([key, val]) => `- ${key}: correct value is "${val}"`)
  .join('\n');
```

**5.2b — Call Claude to analyse the document**

If `document.content` is null or empty: skip Claude call, create one finding of type `decision_card` with issue: "Document has no content — cannot be analysed."

Otherwise call Claude:

```typescript
const prompt = `You are reviewing an investor document for compliance violations and content quality issues.

DOCUMENT NAME: ${document.name}
DOCUMENT TIER: ${document.tier}

PROHIBITED VALUES (must never appear in any investor-facing document):
${prohibitedList}

CANONICAL VALUES (correct values that must be used):
${canonicalList}

DOCUMENT CONTENT:
${document.content}

Analyse this document and return a JSON array of findings. Return JSON only — no other text.

Each finding must be:
{
  "finding_type": "auto_fix" | "decision_card",
  "issue_description": "specific description of the problem",
  "original_text": "the exact text that contains the problem (max 100 chars)",
  "proposed_fix": "the exact replacement text or specific suggestion",
  "severity": "HIGH" | "MEDIUM" | "LOW"
}

Rules for classification:
- "auto_fix": the document contains a prohibited value or an incorrect canonical figure that has a known correct replacement. The fix is a direct text substitution. No judgement required.
- "decision_card": a content quality issue that requires operator judgement — missing framing, structural gaps, tone problems, outdated arguments, missing belief alignment.

Rules for findings:
- Only report real issues. If the document is compliant and high quality, return an empty array [].
- For auto_fix: original_text must be the exact string to replace. proposed_fix must be the exact replacement string.
- For decision_card: proposed_fix must be a specific, short, actionable suggestion — not a vague comment.
- Do not invent issues. Do not report issues not supported by the document content.
- Maximum 10 findings per document.
- Order findings: auto_fix first, then decision_card by severity (HIGH → MEDIUM → LOW).`;

const response = await anthropic.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 8192,
  messages: [{ role: 'user', content: prompt }],
});

const rawText = response.content
  .filter(b => b.type === 'text')
  .map(b => b.text)
  .join('');

const jsonMatch = rawText.match(/\[[\s\S]*\]/);
if (!jsonMatch) throw new Error('No JSON array found in Claude response');
const findings = JSON.parse(jsonMatch[0]) as Array<{
  finding_type: string;
  issue_description: string;
  original_text?: string;
  proposed_fix?: string;
  severity: string;
}>;
```

**5.2c — Persist findings**

Find the task for this document:
```typescript
const task = tasks.find(t => t.linked_document_id === document.id);
if (!task) continue; // should not happen but guard anyway
```

Calculate sort_order: `(document.tier * 100) + index` — Tier 1 docs get sort_order 100–199, Tier 2 get 200–299, Tier 3 get 300–399.

Insert findings:
```typescript
if (findings.length > 0) {
  await db.insert(workQueueFindingsTable).values(
    findings.map((f, i) => ({
      session_id: sessionId,
      task_id: task.id,
      document_id: document.id,
      document_name: document.name,
      document_tier: document.tier,
      finding_type: f.finding_type,
      issue_description: f.issue_description,
      original_text: f.original_text || null,
      proposed_fix: f.proposed_fix || null,
      status: 'PENDING',
      sort_order: (document.tier * 100) + i,
    }))
  );
}
```

If findings is empty array: no findings inserted for this document. Task is implicitly clean.

**5.2d — Update session progress**
```typescript
await db.update(workQueueSessionsTable)
  .set({ analysed_tasks: sql`analysed_tasks + 1` })
  .where(eq(workQueueSessionsTable.id, sessionId));
```

**5.2e — After all documents processed**

Count totals:
```typescript
const allFindings = await db.select().from(workQueueFindingsTable)
  .where(eq(workQueueFindingsTable.session_id, sessionId));

const autoFixCount = allFindings.filter(f => f.finding_type === 'auto_fix').length;
const cardCount = allFindings.filter(f => f.finding_type === 'decision_card').length;
```

Update session to READY:
```typescript
await db.update(workQueueSessionsTable)
  .set({
    status: 'READY',
    cards_total: cardCount,
    auto_fixed_count: 0, // auto_fix findings exist but haven't been applied yet
    completed_at: new Date(),
  })
  .where(eq(workQueueSessionsTable.id, sessionId));
```

Note: `auto_fixed_count` in the session tracks *applied* auto-fixes, not *found* ones. Found auto-fixes are in the findings table with status PENDING.

### 5.1 — GET /work-queue/cards

Returns pending decision_card findings for the current READY session, ordered by sort_order ascending.

Logic:
1. Find current READY session (same query as GET /status)
2. If no READY session: return `{ cards: [], session: null }`
3. Query findings: `finding_type = 'decision_card'` AND `status = 'PENDING'` AND `session_id = session.id`, ordered by `sort_order ASC`
4. Return `{ cards: WorkQueueFinding[], session: WorkQueueSession, total_remaining: number }`

Error: 500 `{ error: 'Failed to fetch cards' }`

---

### 5.2 — POST /work-queue/auto-fix

Applies all pending auto_fix findings for the current READY session.

Logic:
1. Find current READY session. If none: 404 `{ error: 'No active session' }`
2. Fetch all findings where `finding_type = 'auto_fix'` AND `status = 'PENDING'` AND `session_id = session.id`
3. If none: return `{ applied: 0, message: 'No auto-fixes pending' }`
4. For each auto_fix finding:

   a. Fetch the document from `documentsTable`
   b. If `original_text` and `proposed_fix` are both set AND `document.content` contains `original_text`:
      - Replace first occurrence of `original_text` with `proposed_fix` in `document.content`
      - Update `documentsTable` set `content = newContent`, `review_state = 'CLEAN'`
      - Mark the linked task as Done: `db.update(tasksTable).set({ status: 'Done' })`
      - Write changelog: `action: 'AUTO_FIX_APPLIED'`, `triggered_by: 'agent'`, details: finding.issue_description
      - Call `propagateFromDocument(document.id)` then `createReviewTasksForPropagation(result.targets)`
      - Update cascaded_count on session: `sql\`cascaded_count + result.targets.length\``
      - Mark finding: `status = 'AUTO_FIXED'`, `resolved_at = new Date()`
   
   c. If `original_text` not found in document content OR `original_text`/`proposed_fix` null:
      - Mark finding as `status = 'FAILED'`, add note to issue_description
      - Do NOT mark the task as Done
      - Do NOT trigger propagation

5. Update session: `auto_fixed_count = number of successfully applied fixes`
6. Return `{ applied: N, failed: M, session: updatedSession }`

Error: 500 `{ error: 'Failed to apply auto-fixes' }`

---

### 5.3 — POST /work-queue/cards/:findingId/accept

Applies a single decision card fix and resolves the task.

Logic:
1. Fetch finding by `findingId`. If not found: 404
2. If finding.status !== 'PENDING': 400 `{ error: 'Finding already resolved' }`
3. Fetch the document from `documentsTable`
4. If `proposed_fix` is set AND `original_text` is set AND document.content contains `original_text`:
   - Replace first occurrence of `original_text` with `proposed_fix` in document.content
   - Update document: `content = newContent`, `review_state = 'CLEAN'`
5. If `original_text` is null (structural issue, no direct text replacement):
   - Update document: `review_state = 'CLEAN'` only (no content change — operator acknowledged the issue)
6. Mark task as Done: `db.update(tasksTable).set({ status: 'Done' }).where(eq(tasksTable.id, finding.task_id))`
7. Write changelog: `action: 'CARD_ACCEPTED'`, `triggered_by: 'operator'`
8. Call `propagateFromDocument(document.id)` then `createReviewTasksForPropagation(result.targets)`
9. Update session: `cards_resolved = sql\`cards_resolved + 1\``, `cascaded_count = sql\`cascaded_count + result.targets.length\``
10. Mark finding: `status = 'ACCEPTED'`, `resolved_at = new Date()`
11. Return `{ finding: updatedFinding, cascaded: result.targets.length, session: updatedSession }`

Error: 500 `{ error: 'Failed to accept card' }`

---

### 5.4 — POST /work-queue/cards/:findingId/skip

Skips a finding with a reason.

Logic:
1. Fetch finding by `findingId`. If not found: 404
2. If finding.status !== 'PENDING': 400 `{ error: 'Finding already resolved' }`
3. Body: `{ reason: string }` — must be one of: `'Not relevant'`, `'Needs more context'`, `'Defer'`. If invalid: 400 `{ error: 'Invalid skip reason' }`
4. Mark finding: `status = 'SKIPPED'`, `skip_reason = reason`, `resolved_at = new Date()`
5. Update session: `cards_skipped = sql\`cards_skipped + 1\``
6. Do NOT mark the task as Done. Do NOT trigger propagation. Task remains Open.
7. Return `{ finding: updatedFinding, session: updatedSession }`

Error: 500 `{ error: 'Failed to skip card' }`

---

### 5.5 — GET /work-queue/summary

Returns the session summary for the completed session.

Logic:
1. Find the most recent session (any status), ordered by started_at DESC, limit 1
2. If none: return `{ summary: null }`
3. Count findings by status for that session
4. Return:
```json
{
  "summary": {
    "session": WorkQueueSession,
    "auto_fixed": number,
    "cards_accepted": number,
    "cards_skipped": number,
    "cascaded": number,
    "still_open": number
  }
}
```
Where `still_open` = count of findings with status PENDING or FAILED for this session.

Error: 500 `{ error: 'Failed to fetch summary' }`

---

### 5.6 — Session completion

After each accept or skip action: check if all findings for the session are resolved (no PENDING findings remain). If so: update session status to COMPLETE.

```typescript
const pendingCount = await db.select({ count: sql<number>`count(*)` })
  .from(workQueueFindingsTable)
  .where(and(
    eq(workQueueFindingsTable.session_id, finding.session_id),
    eq(workQueueFindingsTable.status, 'PENDING')
  ));

if (pendingCount[0].count === 0) {
  await db.update(workQueueSessionsTable)
    .set({ status: 'COMPLETE', completed_at: new Date() })
    .where(eq(workQueueSessionsTable.id, finding.session_id));
}
```
