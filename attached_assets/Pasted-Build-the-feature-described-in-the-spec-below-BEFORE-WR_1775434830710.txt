## Section 1 — Feature

Adds a "Sync from Google Sheet" tab to the existing Import Content page (`/import`). The operator pastes a Google Sheet URL, clicks Sync Now, and the system reads the sheet via the existing Replit Connectors SDK — creating new leads and storing transcript text for rows not yet imported. Each synced lead and transcript then appears in the Leads list ready for manual analysis via the Recommendation Engine.

---

## Section 2 — Why this is necessary

Call logs and transcripts are currently tracked in a Google Sheet (structured with Lead, Call Transcript, Call Recording columns) and imported into the platform manually. There is no automated path from the sheet to the leads database — transcripts must be uploaded one by one to the Recommendation Engine. This build creates a direct sync path that eliminates manual data entry and makes every transcript immediately available for analysis.

### 5.4 — API: sheet sync router

File: `artifacts/api-server/src/routes/sheet-sync/index.ts` (create new file)

Create an Express Router. Routes in this exact order:

**POST /sheet-sync/sync**

Body: `{ sheet_url: string }`

Validate: `sheet_url` must be non-empty and start with `https://docs.google.com/spreadsheets/`. If invalid: 400 `{ error: 'Invalid Google Sheet URL' }`.

Behaviour:

1. Create session in `sheetSyncSessionsTable` with `status: 'RUNNING'`, `sheet_url`, `created_at: now()`.

2. Extract spreadsheet ID:
```typescript
const match = sheet_url.match(/\/spreadsheets\/d\/([a-zA-Z0-9-_]+)/);
if (!match) return 400 { error: 'Could not extract spreadsheet ID from URL' };
const spreadsheetId = match[1];
```

3. Read the sheet via Replit Connectors SDK — Google Sheets API v4. ⚠️ STOP HERE AND RAISE BLOCKER about connector service name before implementing this call.

Expected response shape: `{ values: string[][] }` — row 0 is header (skip). Column 0 = Lead name, column 1 = Call Transcript, column 2 = Call Recording (ignore).

If connector call fails: update session to `FAILED`, `error_message`, `completed_at`. Return 502 `{ error: 'Could not read Google Sheet', session_id }`.

4. Load all existing leads: `const allLeads = await db.select().from(leadsTable);`

5. For each data row (skip header, skip empty name rows):

   a. Extract `leadName = row[0]?.trim()`, `transcriptText = row[1]?.trim()`. If either empty: increment `rows_failed`, continue.

   b. Run `fuzzyMatchLeads(leadName, allLeads)`. Take top result if score >= 0.4.

   c. Match found AND `match.lead.transcript_text` already populated → skip, increment `leads_skipped`.

   d. Match found AND `match.lead.transcript_text` is null/empty → update lead's `transcript_text`, write changelog `TRANSCRIPT_SYNCED_FROM_SHEET`, increment `leads_updated`.

   e. No match (score < 0.4) → create new lead:
   ```typescript
   id: `lead_${randomUUID().slice(0, 8)}`
   name: leadName
   source: 'sheet_sync'
   transcript_text: transcriptText
   pipeline_stage: 'Outreach'
   ```
   Write changelog `LEAD_CREATED_FROM_SHEET`, increment `leads_created`.

6. Determine final status: all pass → `COMPLETE`, some fail → `PARTIAL`, all fail → `FAILED`. Update session record. Return 200 with session summary.

Wrap Steps 2–6 in try/catch: on error, update session to `FAILED`, return 500.

**GET /sheet-sync/sessions** (declare BEFORE /:id)

Query `sheetSyncSessionsTable` ordered by `created_at` desc, limit 20. Return `{ sessions: [] }`.

**GET /sheet-sync/sessions/:id**

Return single session or 404.

Changelog entries use `triggered_by: 'sheet_sync'` throughout.

File: `artifacts/api-server/src/routes/index.ts` (additive edit)

Add: `router.use('/sheet-sync', sheetSyncRouter)` after tasks router mount.

## Section 6 — Acceptance criteria

AC1. `POST /api/sheet-sync/sync` with a valid Google Sheet URL returns 200 with `{ session_id, status, rows_found, leads_created, leads_updated, leads_skipped, rows_failed }`. Manual test required: use the actual Google Sheet URL, click Sync Now, verify the response fields are populated correctly.

AC2. A new lead is created in `leadsTable` for each sheet row where no fuzzy match exists (score < 0.4). The created lead has `source: 'sheet_sync'`, `pipeline_stage: 'Outreach'`, `transcript_text` populated, and id in `lead_XXXXXXXX` format. Manual test required: add a row with a name not in the leads table, sync, confirm the lead appears in the Leads list.

AC3. A sheet row where a fuzzy match exists (score >= 0.4) and the matched lead has `transcript_text: null` results in that lead's `transcript_text` being updated. No duplicate lead is created. Manual test required: add a row matching an existing lead with no transcript, sync, confirm transcript_text is populated and no duplicate exists.

AC4. A sheet row where a fuzzy match exists and the matched lead already has `transcript_text` populated is skipped — transcript_text is not overwritten, `leads_skipped` increments. Manual test required: sync the same sheet twice; on the second sync, all previously processed rows appear in `leads_skipped`.

AC5. A `sheet_sync_sessions` row is created for every sync call with `sheet_url`, `status`, `rows_found`, `leads_created`, `leads_updated`, `leads_skipped`, `rows_failed`, `created_at`, and `completed_at`. Manual test required: inspect the database after a sync.

AC6. Session status is `COMPLETE` when all rows processed without error, `PARTIAL` when some rows fail, `FAILED` when all rows fail or the sheet is unreadable.

AC7. `GET /api/sheet-sync/sessions` returns 200 with up to 20 sessions ordered by `created_at` descending.

AC8. `GET /api/sheet-sync/sessions/:id` returns the full session object for a valid id, and 404 for an unknown id.

AC9. `POST /api/sheet-sync/sync` with a URL not starting with `https://docs.google.com/spreadsheets/` returns 400 `{ error: 'Invalid Google Sheet URL' }`.

AC10. A changelog entry with `action: 'LEAD_CREATED_FROM_SHEET'` and `triggered_by: 'sheet_sync'` exists for each newly created lead. Manual test required: inspect changelog table after syncing a sheet with new rows.

AC11. A changelog entry with `action: 'TRANSCRIPT_SYNCED_FROM_SHEET'` and `triggered_by: 'sheet_sync'` exists for each lead where `transcript_text` was written. Manual test required: inspect changelog table.

AC12. The "Sync from Google Sheet" tab appears on `/import` alongside "Import Document". The existing import flow is unaffected. Manual test required: navigate to `/import`, confirm both tabs present and switching between them does not affect the other tab's state.

AC13. Clicking "Sync Now" with a valid URL triggers the sync and displays the result card with counts. Manual test required: perform a sync and verify the result card appears.

AC14. The session history table shows the last sync session after a sync completes with correct status and counts. Manual test required: sync, confirm the session appears in the history table.

AC15. `fuzzyMatchLeads` is defined in exactly one place: `artifacts/api-server/src/lib/fuzzyMatch.ts`. The function in `leads/index.ts` is replaced with an import. `GET /api/leads/match?name=X` still returns correct results. Manual test required: call the leads match endpoint and confirm results are unchanged.

AC16. All new lead IDs follow the `lead_XXXXXXXX` format. Manual test required: inspect a synced lead's id field in the database.
