## Section 1 — Feature

This build adds Puppeteer-based PDF export for any document in the Registry,
staleness tracking so the operator can see at a glance whether the current PDF
reflects the current markdown content, and a version history panel surfacing the
existing changelog entries per document. Documents are authored as markdown in
the database; there is currently no way to produce a send-ready, brand-compliant
PDF output from that content.

---

## Section 2 — Why this is necessary

Without this build:
- There is no PDF output from the document registry. Investor-facing documents
  exist only as markdown stored in the database with no way to produce a
  formatted, send-ready file.
- There is no signal to the operator that a document has been edited since its
  last export. A stale PDF could be sent to a lead without any warning.
- Document version history exists in the changelog table but is invisible — there
  is no UI surface that shows what changed, when, and at which version.

### Step 3 — Backend: PDF service

Create `artifacts/api-server/src/lib/pdfService.ts`.

This file must export one function:

```typescript
export async function generatePdf(document: {
  id: string;
  name: string;
  file_code: string;
  tier: string;
  version: number;
  content: string;
}): Promise<{ filePath: string }>
```

**Behaviour:**

1. Convert `document.content` (markdown) to HTML using `marked`. The `marked`
   package must be installed: `pnpm --filter api-server add marked`.

2. Wrap the HTML in the Design Bible template (see template spec below).

3. Launch a Puppeteer browser instance with `args: ['--no-sandbox']`. Reuse a
   module-level browser instance if one is already running; do not launch a new
   browser per request.

4. Open a new page, set the HTML content, wait for fonts to load
   (`waitUntil: 'networkidle0'`).

5. Print to PDF:
   ```typescript
   await page.pdf({
     format: 'A4',
     printBackground: true,
     margin: { top: '25mm', right: '25mm', bottom: '25mm', left: '25mm' },
   });
   ```

6. Write the PDF buffer to disk at:
   `/artifacts/exports/pdf/{document.id}_{document.version}.pdf`

   Create the directory if it does not exist (`fs.mkdirSync(..., { recursive: true })`).
   Overwrite any existing file at that path — re-exporting the same version
   replaces the prior file.

7. Close the page (not the browser).

8. Return `{ filePath }`.

9. On any error: close the page if open, do not write to disk, throw the error
   so the calling route can return 500.

**Design Bible HTML template:**

The template must be constructed as a single self-contained HTML string with
all CSS inline or in a `<style>` block. Inter font must be embedded using a
local `@font-face` declaration — do not load from Google Fonts CDN or any
external URL. The font files must be bundled with the server at a known path
(e.g. `artifacts/api-server/src/assets/fonts/Inter-*.woff2`) and referenced
via absolute path in the `@font-face` src.

If Inter font files are not yet present in the repository, download them from
the Inter GitHub release (https://github.com/rsms/inter/releases) and add
them to `artifacts/api-server/src/assets/fonts/`. Do not rely on system fonts
or CDN for PDF rendering.

### Step 4 — Backend: Export endpoints

Add two routes to the documents router. These must be declared **before** any
`:id`-parameterised routes within the same router to prevent route swallowing.

Confirm the exact documents router file at Step 1 scan. It is likely
`artifacts/api-server/src/routes/documents/index.ts`.

**POST /api/documents/:id/export/pdf**

```
1. Fetch document by id from documentsTable.
   If not found: return 404 { error: "Document not found" }.

2. Call pdfService.generatePdf(document).
   If generatePdf throws: return 500 { error: error.message }.
   Do not update any DB fields on failure.

3. On success:
   a. Update documentsTable set:
        pdf_exported_at = NOW()
        pdf_file_path = filePath
      where id = document.id
   b. Write changelog entry:
        action: 'PDF_EXPORTED'
        document_id: document.id
        metadata: { version: document.version, file_path: filePath }
   c. Return 200 { success: true, file_path: filePath, exported_at: <timestamp> }

Order of writes: file must be written before DB is updated.
DB must be updated before changelog is written.
If DB update fails after file write: log the error, return 500.
The file on disk is the authoritative artifact; a missing DB record is
repairable by re-exporting.
```

**GET /api/documents/:id/export/pdf**

```
1. Fetch document by id from documentsTable.
   If not found: return 404 { error: "Document not found" }.

2. If pdf_file_path is null: return 404 { error: "No PDF generated yet" }.

3. Check that the file exists at pdf_file_path using fs.existsSync().
   If file is missing: return 410 { error: "PDF file missing on disk — re-export required" }.

4. Stream the file:
   Content-Type: application/pdf
   Content-Disposition: attachment; filename="{file_code}_v{version}.pdf"
   Use fs.createReadStream(pdf_file_path).pipe(res).
```

**OpenAPI spec** — add both endpoints to `lib/api-spec/openapi.yaml`.
Run codegen after: `pnpm --filter @workspace/api-spec run g`.

### Step 5 — Backend: content_updated_at writes

Every code path that writes to `documents.content` must also set
`content_updated_at = NOW()` in the same DB update.

Affected locations — confirm exact files at Step 1 scan:

- `POST /import/quick-update` handler — add `content_updated_at: new Date()`
  to the update object alongside the content write.
- `PATCH /documents/:id` handler — if `content` is present in the request
  body, add `content_updated_at: new Date()` to the update.
- Any generation route that persists content to documentsTable — add
  `content_updated_at: new Date()` to those update calls.

**Rule:** `content_updated_at` must never be set by `POST /documents/:id/export/pdf`
or `GET /documents/:id/export/pdf`. It tracks content changes only.

### Step 6 — Frontend: Registry page updates

File: `artifacts/unlock-intel/src/pages/registry.tsx` (confirm at scan).

**PDF status badge**

Compute staleness from the document record fields returned by the existing
documents list / detail API. Add a badge component to each document row or
expanded detail panel:

| Condition | Badge |
|---|---|
| `pdf_exported_at` is null | Grey — Clock icon — "No PDF" |
| `pdf_exported_at < content_updated_at` | Amber — AlertTriangle icon — "PDF outdated" |
| `pdf_exported_at >= content_updated_at` | Green — CheckCircle icon — "PDF current" |

Use Lucide icons: `Clock`, `AlertTriangle`, `CheckCircle2`. Use Shadcn Badge
component with variant prop. Colour values: grey = `#888888`, amber = `#F59E0B`,
green = `#008655` (Text Green — passes WCAG AA on white).

**Action buttons**

In the document detail panel or expanded row:

- **Generate PDF** button — always visible for CURRENT documents.
  On click: `POST /api/documents/:id/export/pdf`.
  Show loading spinner on the button during the request.
  On success: invalidate the document query to refresh badge state.
  On error: show a toast/alert with the error message.

- **Download PDF** button — visible only when `pdf_file_path` is not null.
  On click: navigate to `GET /api/documents/:id/export/pdf` (browser will
  handle the file download via Content-Disposition).

Both buttons use Shadcn Button component. "Generate PDF" uses the `FileOutput`
Lucide icon. "Download PDF" uses the `Download` Lucide icon.

**Version history panel**

Collapsible accordion section below document metadata. On expand, fetch:

```
GET /api/changelog?document_id={id}&limit=20&order=desc
```

Confirm the exact changelog endpoint at Step 1 scan — use whatever exists.

Display each changelog entry as a row:

```
v{version}  |  {action label}  |  {timestamp formatted as "6 Apr 2026, 14:32"}  |  {word_count delta or "—"}
```

Word count delta: if `metadata.word_count` is present on the changelog entry,
compute delta from prior entry and show as `+142 words` or `−23 words`. If
absent, show `—`. Do not crash if metadata is null or malformed.

Maximum 20 entries. No pagination required in this build.

```

      with { success: true, file_path, exported_at }. A PDF file exists at

      application/pdf and Content-Disposition: attachment; filename containing
      file_code and version. Browser receives the file as a download.

      generated yet" } when pdf_file_path is null on the document record.

AC5.  GET /api/documents/:id/export/pdf returns 410 { error: "PDF file

      the DB but the file does not exist at that path on disk.

AC6.  After a Quick Update to a document's content field, content_updated_at
      is set to a timestamp later than its prior value. If a PDF had been
      previously generated, the Registry badge shows "PDF outdated" (amber).

AC8.  A document where pdf_exported_at >= content_updated_at shows the
      "PDF current" (green) badge.

      file at the same path. pdf_exported_at and pdf_file_path are updated.
      The prior file no longer exists at the path before overwrite; after
```

## Section 7 — Constraints

```
- artifacts/api-server/src/lib/propagation.ts: must not be modified.
  PDF export does not affect propagation logic.

- artifacts/api-server/src/lib/generationEngine.ts: must not be modified.
  PDF generation is entirely separate from the LLM generation pipeline.

- lib/api-spec/openapi.yaml: must be updated to include both new endpoints
  before codegen is run. Do not run codegen without updating the spec first.

- lib/db/src/schema/index.ts: add the barrel export for any new schema
  file. Do not remove existing exports.

- Route ordering: within the documents router, POST /documents/:id/export/pdf
  and GET /documents/:id/export/pdf must be declared before any route
  matching /documents/:id (i.e., before the general PATCH/GET/:id routes)
  to prevent :id swallowing the /export/pdf path segment.

- content_updated_at: must never be set by the PDF export endpoints.
  It is set only by routes that write to the content field.

- pdf_exported_at and pdf_file_path: must never be set by routes that
  write to the content field. Each timestamp tracks a distinct event.

- Backward compatibility: the three new schema columns are nullable with
  no default. All existing document records will have null in these fields
  until content is next written or a PDF is exported. No existing behaviour
  changes. The documents list and detail APIs continue to function unchanged;
  the new fields are additive to their response shapes.

- Font assets: Inter font files must be bundled with the server at a known
  path. Do not load fonts from any external CDN at PDF render time.

- Puppeteer browser instance: launch once at the module level and reuse
  across requests. Do not launch a new browser per PDF export request.
  Handle browser crash/disconnect by relaunching on next request.
```

   remove the two export routes from the documents router.

3. **Modified routes:** Revert the `content_updated_at = NOW()` additions
   to quick-update, PATCH /documents/:id, and any generation routes. These

4. **Frontend:** Revert the Registry page to remove badge, buttons, and
   version history panel. No new pages were added.

5. **Disk:** Delete `/artifacts/exports/pdf/` directory and contents.

6. **OpenAPI + codegen:** Revert openapi.yaml changes and re-run codegen.

7. **Puppeteer:** Uninstall with `pnpm --filter api-server remove puppeteer`.
   Remove Inter font assets from `artifacts/api-server/src/assets/fonts/`.

No external state (no third-party services, no lead data, no ACU data) is
affected by this build. Rollback is fully reversible.
