## Section 1 — Feature

Adds a Document Health Check page at `/document-health` under the
CONTENT section of the sidebar. The operator clicks "Run Health Check"
and the system analyses every CURRENT document across eight dimensions:
identity completeness, targeting, belief alignment, compliance,
propagation integrity, pipeline stage fit, content completeness, and
delivery record. Results persist to a `document_health_sessions` table.
The UI shows a scorecard — one row per document with ✓/⚠/✗ per
dimension — with expandable rows for detail and direct fix actions.

---

## Section 2 — Why this is necessary

The document library has no systematic quality assurance beyond the
Work Queue (which only catches compliance violations). There is no way
to see at a glance which documents are missing persona targeting, have
no belief mapping, are disconnected from the propagation cascade, or
have never been sent. The health check surfaces all of these gaps in
one view so the operator can prioritise fixes.

### 5.2 — New router: document-health

File: `artifacts/api-server/src/routes/document-health/index.ts` (create)

Two endpoints:

---

**POST /document-health/run**

Starts a health check run. Returns 202 immediately.
Runs analysis in background via setImmediate.

Concurrency guard: if a session with status RUNNING exists, return it
with `{ session, already_running: true }`.

Logic (synchronous, before setImmediate):
1. Create session with status RUNNING
2. Return 202 `{ session, already_running: false }`

Logic (background, in setImmediate):
3. Fetch all CURRENT documents
4. Fetch all active compliance constants (split into prohibited/canonical)
5. Fetch all belief_registry rows
6. Fetch all lead send_logs (parse documents_sent arrays)
7. Fetch all open Review tasks (for compliance dimension)
8. For each document, run all 8 dimension checks (see 5.3)
9. Insert a document_health_scores row per document
10. Update session: status COMPLETE, counts, completed_at

Error handling: on any unrecovered error, set session status FAILED
with error_message.

---

**GET /document-health/latest**

Returns the most recent session and all its scores.

```typescript
// Find most recent session
const session = await db.select().from(documentHealthSessionsTable)
  .orderBy(desc(documentHealthSessionsTable.started_at))
  .limit(1);

if (!session[0]) return res.json({ session: null, scores: [] });

// Fetch scores for that session
const scores = await db.select().from(documentHealthScoresTable)
  .where(eq(documentHealthScoresTable.session_id, session[0].id))
  .orderBy(
    asc(documentHealthScoresTable.document_tier),
    asc(documentHealthScoresTable.document_name)
  );

return res.json({ session: session[0], scores });
```

### 5.3 — Eight dimension checks

Run these checks per document inside the background analysis.
Each check produces a status (PASS/WARN/FAIL) and an issues array.

Status rules:
- PASS: no issues
- WARN: minor issue — document still functional but suboptimal
- FAIL: document cannot fulfil its purpose on this dimension

Overall document status:
- PASS: all dimensions PASS
- WARN: at least one WARN, no FAIL
- FAIL: at least one FAIL

---

**Dimension 1 — Identity**

Check: tier, category, name, file_code, description all populated.

```typescript
const issues = [];
if (!doc.tier) issues.push({ field: 'tier', message: 'Missing tier' });
if (!doc.category) issues.push({ field: 'category', message: 'Missing category' });
if (!doc.description || doc.description.trim() === '') issues.push({ field: 'description', message: 'Missing description' });
if (!doc.file_code) issues.push({ field: 'file_code', message: 'Missing file_code' });

const status = issues.length === 0 ? 'PASS' : issues.length <= 1 ? 'WARN' : 'FAIL';
```

---

**Dimension 2 — Targeting**

Check: persona_relevance and pipeline_stage_relevance both non-empty.

```typescript
const issues = [];
const personas = doc.persona_relevance as string[] || [];
const stages = doc.pipeline_stage_relevance as string[] || [];

if (personas.length === 0) issues.push({ field: 'persona_relevance', message: 'No persona targeting — document will never be recommended' });
if (stages.length === 0) issues.push({ field: 'pipeline_stage_relevance', message: 'No stage targeting — document will never be recommended' });

const status = issues.length === 0 ? 'PASS' : issues.some(i => i.field === 'persona_relevance' && i.field === 'pipeline_stage_relevance') ? 'FAIL' : 'WARN';
// FAIL if both missing, WARN if one missing
const status = issues.length === 0 ? 'PASS' : issues.length === 2 ? 'FAIL' : 'WARN';
```

---

**Dimension 3 — Belief Alignment**

Check: belief_targets non-empty. Check each belief_id in belief_targets
exists in beliefRegistry.

```typescript
const issues = [];
const targets = doc.belief_targets as Array<{belief_id: string, state_from: string, state_to: string}> || [];

if (targets.length === 0) {
  issues.push({ field: 'belief_targets', message: 'No belief mapping — document cannot contribute to belief progression' });
} else {
  for (const target of targets) {
    const belief = beliefRegistry.find(b => b.id === target.belief_id);
    if (!belief) issues.push({ field: 'belief_targets', message: `References non-existent belief: ${target.belief_id}` });
    if (belief && belief.policy_status === 'blocked_pending_legal') {
      issues.push({ field: 'belief_targets', message: `Targets legally blocked belief: ${target.belief_id}` });
    }
  }
}

const status = issues.length === 0 ? 'PASS' :
  issues.some(i => i.message.includes('non-existent')) ? 'FAIL' : 'WARN';
```

---

**Dimension 4 — Compliance**

Check: review_state, and scan content for prohibited values.

```typescript
const issues = [];

if (doc.review_state === 'REQUIRES_REVIEW') {
  const hasOpenTask = openReviewTasks.some(t => t.linked_document_id === doc.id);
  issues.push({
    field: 'review_state',
    message: `Flagged for review${hasOpenTask ? ' (task exists)' : ' (NO open task — invisible to Work Queue)'}`,
    has_task: hasOpenTask,
  });
}

// Scan content for prohibited values
if (doc.content) {
  for (const prohibited of prohibitedValues) {
    if (doc.content.toLowerCase().includes(prohibited.value.toLowerCase())) {
      issues.push({
        field: 'content',
        message: `Contains prohibited value: "${prohibited.value}" — ${prohibited.prohibited_reason || 'must not appear'}`,
        prohibited_value: prohibited.value,
      });
    }
  }
}

const status = issues.length === 0 ? 'PASS' :
  issues.some(i => i.field === 'content') ? 'FAIL' :
  'WARN';
```

---

**Dimension 5 — Propagation**

Check: Tier 2/3 documents should have upstream_dependencies.
Check: upstream_dependencies reference CURRENT documents.

```typescript
const issues = [];
const upstreamIds = doc.upstream_dependencies as string[] || [];

if (doc.tier > 1 && upstreamIds.length === 0) {
  issues.push({
    field: 'upstream_dependencies',
    message: `Tier ${doc.tier} document has no upstream dependencies — changes to Tier 1 documents will not cascade to this document`,
  });
}

for (const upId of upstreamIds) {
  const upDoc = allDocs.find(d => d.id === upId);
  if (!upDoc) {
    issues.push({ field: 'upstream_dependencies', message: `References non-existent document: ${upId}` });
  } else if (upDoc.lifecycle_status !== 'CURRENT') {
    issues.push({ field: 'upstream_dependencies', message: `References ${upDoc.lifecycle_status} document: ${upDoc.name}` });
  }
}

const status = issues.length === 0 ? 'PASS' :
  issues.some(i => i.message.includes('non-existent')) ? 'FAIL' : 'WARN';
```

---

**Dimension 6 — Content**

Check: content field populated and substantive.

```typescript
const issues = [];

if (!doc.content) {
  issues.push({ field: 'content', message: 'No content — document has nothing to deliver' });
} else if (doc.content.trim().length < 100) {
  issues.push({ field: 'content', message: `Stub content — only ${doc.content.trim().length} characters` });
} else if (doc.content.trim().length < 500) {
  issues.push({ field: 'content', message: `Thin content — only ${doc.content.trim().length} characters` });
}

const status = issues.length === 0 ? 'PASS' :
  issues.some(i => i.message.includes('No content') || i.message.includes('Stub')) ? 'FAIL' : 'WARN';
```

---

**Dimension 7 — Delivery**

Check: has this document ever been sent to a lead?
Parse send_logs from all leads.

```typescript
// allSentDocIds: Set<string> — built before the per-document loop
// by parsing all lead send_logs

const timesSent = allSendEvents.filter(e => e.document_id === doc.id).length;
const uniqueLeads = new Set(allSendEvents.filter(e => e.document_id === doc.id).map(e => e.lead_id)).size;

const issues = [];
if (timesSent === 0) {
  issues.push({
    field: 'delivery',
    message: 'Never sent to any lead',
    times_sent: 0,
    unique_leads: 0,
  });
}

const status = issues.length === 0 ? 'PASS' : 'WARN';
// Never-sent is WARN not FAIL — document may be new or awaiting the right lead
```

Build `allSendEvents` before the document loop:
```typescript
const allLeads = await db.select({ id: leadsTable.id, send_log: leadsTable.send_log }).from(leadsTable);
const allSendEvents: Array<{ document_id: string; lead_id: string; date: string }> = [];

for (const lead of allLeads) {
  const sendLog = lead.send_log as Array<{ send_id: string; date: string; documents_sent: string[] }> || [];
  for (const entry of sendLog) {
    for (const docId of (entry.documents_sent || [])) {
      allSendEvents.push({ document_id: docId, lead_id: lead.id, date: entry.date });
    }
  }
}
```
