## Step 4 — Update the backend to load master context automatically

Now that the master context file exists, update the generate-brief
endpoint so it automatically loads and includes the master context
on every call — without requiring the caller to pass it manually
each time.

Implementation:
- On startup (or on first call), read the master context file from
  the project directory
- Cache it in memory
- Prepend it to the Claude prompt for every generate-brief call,
  before any additional_context supplied in the request body
- If the file is not found, log a warning and proceed without it —
  do not throw

This means future brief generation calls work correctly with full
Unlock context by default, and any additional_context in the request
body is appended after the master context.
