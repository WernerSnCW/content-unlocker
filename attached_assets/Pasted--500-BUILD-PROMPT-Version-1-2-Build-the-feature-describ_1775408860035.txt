## Section 1 — Feature

Adds a POST /documents/:id/quality-score endpoint that evaluates any document
in the registry against a structured six-dimension quality rubric using a single
Claude call. Returns a non-persisted score report with per-dimension scores (0–10),
findings, verdicts (PASS/ADVISORY/FAIL), and an overall score and verdict.
Phase A — backend only. No frontend changes.

---

## Section 2 — Why this is necessary

The existing 19-check QC pipeline confirms compliance. It does not evaluate
quality — strategic alignment, persona fit, structural completeness, or tone.
Documents can pass QC and still be unconvincing or wrongly framed. There is no
mechanism to evaluate this today.

### 5.3 — Call Claude and parse response

Call anthropic.messages.create() with:
- model: "claude-sonnet-4-6"
- max_tokens: 8192
- messages: [{ role: "user", content: promptString }]

Parse response:
- Extract text from first content block
- Apply regex /\{[\s\S]*\}/ to find JSON
- JSON.parse() the match
- If parsing fails: return 500 with { error: "Quality scoring failed — could not parse response" }

Validate parsed object has: dimensions (object, 6 keys), each with score (number),
verdict (string), findings (array). If validation fails: return 500 with
{ error: "Quality scoring failed — malformed response" }

Calculate server-side (override Claude's values):
- overall_score = Math.round(sum of all six dimension scores / 6)
- overall_verdict:
  - If any dimension verdict is "FAIL": "FAIL"
  - Else if any dimension verdict is "ADVISORY": "ADVISORY"
  - Else: "PASS"

### 5.4 — Return response

HTTP 200 with the validated, server-calculated score object including:
- All six dimensions (score, verdict, findings)
- overall_score (server-calculated)
- overall_verdict (server-calculated)
- document_type: document.type
- persona_context: the persona string used
- content_truncated: boolean

## Section 6 — Acceptance criteria

AC1. POST /documents/:id/quality-score with a valid document ID containing
content returns HTTP 200 with JSON body containing dimensions (6 keys),
overall_score (integer), overall_verdict (PASS/ADVISORY/FAIL), document_type,
persona_context, and content_truncated.

AC2. Each dimension contains score (integer 0-10), verdict (PASS/ADVISORY/FAIL),
and findings (array of 1-3 strings). Manual test required: inspect response body.

AC3. overall_score equals the average of the six dimension scores rounded to
nearest integer — calculated server-side.

AC4. overall_verdict is the worst dimension verdict (FAIL > ADVISORY > PASS)
— calculated server-side.

AC5. POST /documents/:id/quality-score with a document that has null or empty
content returns HTTP 400 with { error: "Document has no content to score" }.

AC6. POST /documents/:id/quality-score with a non-existent document ID returns
HTTP 404.

AC7. For a document whose content exceeds 6000 characters, content_truncated
is true in the response.

AC8. The existing POST /documents/:id/qc-rerun endpoint is unchanged and
continues to return its existing response shape. Manual test required.

AC9. No changes to lib/db/src/schema/documents.ts. Manual test required: git diff.

AC10. No changes to artifacts/api-server/src/routes/generation/index.ts.
