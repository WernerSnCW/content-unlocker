## Section 1 — Feature

Fixes three design flaws in the Work Queue system identified by the
Document Intelligence Health Check:

**Flaw 1 — Queue scope depends on tasks existing:** The queue queries
tasks to find documents to analyse. Any REQUIRES_REVIEW document without
an open Review task is permanently invisible. Fix: query documents
directly, auto-create tasks as a side effect, not as a prerequisite.

**Flaw 2 — Task Done → document CLEAN (already partially reverted):**
Marking a Review task Done must never automatically set document
review_state to CLEAN. Only Work Queue resolution (auto-fix applied
successfully, or card accepted) should set CLEAN. These concerns must
be fully decoupled.

**Flaw 3 — Auto-fix reports success regardless of text match:** The
auto-fix flow marks findings as AUTO_FIXED and sets documents to CLEAN
even when the text replacement fails silently. The fix must verify the
replacement actually applied before asserting success.

---

## Section 2 — Why this is necessary

The three flaws together create false confidence:
- 18 REQUIRES_REVIEW documents were invisible to the queue
- Doc 110 was falsely marked CLEAN after a failed auto-fix
- Content_Bank (700) — a Tier 1 document — contained prohibited values
  with no task and no queue visibility
- The task completion flow was asserting document state it had no right
  to assert

These are not edge cases. They are systematic gaps in the core resolution
logic that will recur on every queue run.

## Section 5 — Required behaviour

### 5.1 — Flaw 1 fix: Document-first queue scope

File: `artifacts/api-server/src/routes/work-queue/index.ts`
Location: POST /work-queue/start handler

**Replace the current task-first query with a document-first query:**

**Step 1 — Query documents directly:**
```typescript
// Fetch all CURRENT documents that need review
const requiresReviewDocs = await db.select().from(documentsTable)
  .where(and(
    eq(documentsTable.lifecycle_status, 'CURRENT'),
    eq(documentsTable.review_state, 'REQUIRES_REVIEW')
  ));

if (requiresReviewDocs.length === 0) {
  // No documents need review — create COMPLETE session with 0 tasks
  const session = await db.insert(workQueueSessionsTable).values({
    status: 'COMPLETE',
    total_tasks: 0,
    analysed_tasks: 0,
    completed_at: new Date(),
  }).returning();
  return res.status(200).json({ session: session[0], already_running: false });
}
```

**Step 2 — Ensure each document has an open Review task:**
```typescript
// For each document, find or create an open Review task
const documentTaskMap: Record<string, string> = {}; // document_id → task_id

for (const doc of requiresReviewDocs) {
  const existing = await db.select().from(tasksTable)
    .where(and(
      eq(tasksTable.linked_document_id, doc.id),
      eq(tasksTable.type, 'Review'),
      eq(tasksTable.status, 'Open')
    )).limit(1);

  if (existing[0]) {
    documentTaskMap[doc.id] = existing[0].id;
  } else {
    // Create task for this document
    const newTask = await db.insert(tasksTable).values({
      id: randomUUID(),
      title: `Review: ${doc.name}`,
      status: 'Open',
      type: 'Review',
      linked_document_id: doc.id,
    }).returning();
    documentTaskMap[doc.id] = newTask[0].id;
  }
}
```

**Step 3 — Create session with correct total_tasks count:**
```typescript
const session = await db.insert(workQueueSessionsTable).values({
  status: 'ANALYSING',
  total_tasks: requiresReviewDocs.length,
  analysed_tasks: 0,
}).returning();
const sessionId = session[0].id;

// Return 202 immediately
res.status(202).json({ session: session[0], already_running: false });

// Run analysis in background
setImmediate(async () => {
  try {
    await runAnalysis(sessionId, requiresReviewDocs, documentTaskMap, prohibitedValues, canonicalValues);
  } catch (err) {
    await db.update(workQueueSessionsTable)
      .set({ status: 'FAILED', error_message: String(err), completed_at: new Date() })
      .where(eq(workQueueSessionsTable.id, sessionId));
  }
});
```

**Update runAnalysis signature** to accept `documentTaskMap` instead of
deriving the task from a tasks array:

```typescript
async function runAnalysis(
  sessionId: string,
  documents: Array<typeof documentsTable.$inferSelect>,
  documentTaskMap: Record<string, string>, // document_id → task_id
  prohibitedValues: typeof complianceConstantsTable.$inferSelect[],
  canonicalValues: Record<string, string>
): Promise<void>
```

Inside runAnalysis, replace the task lookup with:
```typescript
const taskId = documentTaskMap[document.id];
if (!taskId) continue; // should not happen
```

**Compliance constants fetch** stays in POST /work-queue/start before
the setImmediate call — same as before.

### 5.3 — Flaw 3 fix: Verified auto-fix with content diff check

File: `artifacts/api-server/src/routes/work-queue/index.ts`
Location: POST /work-queue/auto-fix handler, per-finding loop

**Replace the current auto-fix application logic with a verified replacement:**

Current problematic flow:
1. Check if `original_text` exists in content ✓
2. Replace `original_text` with `proposed_fix`
3. Mark finding AUTO_FIXED ← no verification that content actually changed
4. Set document to CLEAN ← no verification

**New verified flow:**

```typescript
// Step 1: Check original_text exists in content
if (!document.content || !finding.original_text || !finding.proposed_fix) {
  // Cannot apply — mark FAILED
  await db.update(workQueueFindingsTable)
    .set({
      status: 'FAILED',
      issue_description: finding.issue_description + ' [AUTO-FIX FAILED: missing content, original_text, or proposed_fix]',
      resolved_at: new Date(),
    })
    .where(eq(workQueueFindingsTable.id, finding.id));
  failedCount++;
  continue;
}

if (!document.content.includes(finding.original_text)) {
  // Text not found — mark FAILED
  await db.update(workQueueFindingsTable)
    .set({
      status: 'FAILED',
      issue_description: finding.issue_description + ' [AUTO-FIX FAILED: original text not found in document]',
      resolved_at: new Date(),
    })
    .where(eq(workQueueFindingsTable.id, finding.id));
  failedCount++;
  continue;
}

// Step 2: Apply replacement
const newContent = document.content.replace(finding.original_text, finding.proposed_fix);

// Step 3: CRITICAL — verify content actually changed
if (newContent === document.content) {
  // Replacement produced no change — do NOT mark as fixed
  await db.update(workQueueFindingsTable)
    .set({
      status: 'FAILED',
      issue_description: finding.issue_description + ' [AUTO-FIX FAILED: replacement produced no change in content]',
      resolved_at: new Date(),
    })
    .where(eq(workQueueFindingsTable.id, finding.id));
  failedCount++;
  continue;
}

// Step 4: Persist the verified new content
await db.update(documentsTable)
  .set({ content: newContent })
  .where(eq(documentsTable.id, document.id));

// Step 5: Verify the fix is now in the document
const verifyDoc = await db.select({ content: documentsTable.content })
  .from(documentsTable)
  .where(eq(documentsTable.id, document.id))
  .limit(1);

const fixVerified = verifyDoc[0]?.content?.includes(finding.proposed_fix) ?? false;

if (!fixVerified) {
  // Write succeeded but verification failed — mark FAILED, do not set CLEAN
  await db.update(workQueueFindingsTable)
    .set({
      status: 'FAILED',
      issue_description: finding.issue_description + ' [AUTO-FIX FAILED: content written but proposed_fix not found on re-read]',
      resolved_at: new Date(),
    })
    .where(eq(workQueueFindingsTable.id, finding.id));
  failedCount++;
  continue;
}

// Step 6: Only now — mark document CLEAN and task Done
await db.update(documentsTable)
  .set({ review_state: 'CLEAN' })
  .where(eq(documentsTable.id, document.id));

await db.update(tasksTable)
  .set({ status: 'Done' })
  .where(eq(tasksTable.id, finding.task_id));

// Step 7: Write changelog
await db.insert(changelogTable).values({
  id: randomUUID(),
  action: 'AUTO_FIX_APPLIED',
  details: `Fixed: ${finding.issue_description}. Replaced: "${finding.original_text}" → "${finding.proposed_fix}"`,
  triggered_by: 'agent',
});

// Step 8: Propagation
const result = await propagateFromDocument(document.id);
await createReviewTasksForPropagation(result.targets);
await db.update(workQueueSessionsTable)
  .set({ cascaded_count: sql`cascaded_count + ${result.targets.length}` })
  .where(eq(workQueueSessionsTable.id, sessionId));

// Step 9: Mark finding AUTO_FIXED only after all of the above
await db.update(workQueueFindingsTable)
  .set({ status: 'AUTO_FIXED', resolved_at: new Date() })
  .where(eq(workQueueFindingsTable.id, finding.id));

appliedCount++;
```
