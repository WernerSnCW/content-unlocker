## Section 1 — Feature

Adds the database foundation for the investor intelligence and belief state system. Four new tables are created (belief_registry, lead_beliefs, belief_transitions, lead_intelligence), the documents table gains a `belief_targets` column, and the leads table `notes` field is migrated from jsonb to a plain text field. The 22 beliefs are seeded into the registry on server startup.

---

## Section 2 — Why this is necessary

Without this schema, no belief state can be tracked per lead, no intelligence record can be persisted, and no document can be mapped to the beliefs it targets. All subsequent phases (010b–010d) depend on these tables existing and the 22 beliefs being seeded. The current `notes` field on leads is JSONB but unused — converting it to text now avoids a more complex migration later when the frontend starts writing to it.

File: `lib/db/src/schema/leads.ts` (edit)

**Strategy: Option A — rename existing field, add new field.**

Step 1: Rename the existing `notes` jsonb column to `notes_legacy`:
```typescript
notes_legacy: jsonb('notes_legacy').default([]),
```

Step 2: Add a new `notes` text column (nullable):
```typescript
notes: text('notes'),
```

Rationale: preserves any existing jsonb data in `notes_legacy` while giving the new feature a clean text field. The existing API returns `notes` — after this change, the old jsonb data is in `notes_legacy` and the new text field is `notes`. Phase 010b will update the API response to return the new text `notes` field.

Run `drizzle-kit push` after this change.

```typescript
export const leadBeliefsTable = pgTable('lead_beliefs', {
  id: text('id').primaryKey().$defaultFn(() => randomUUID()),
  lead_id: text('lead_id').notNull().references(() => leadsTable.id),
  belief_id: text('belief_id').notNull().references(() => beliefRegistryTable.id),
  state: text('state').notNull().default('UNKNOWN'),
  // Canonical values: UNKNOWN / ABSENT / PARTIAL / ESTABLISHED / BLOCKED
  investor_relevance: text('investor_relevance').default('standard'),
  // high / standard / low / not_applicable
  relevance_rationale: text('relevance_rationale'),
  established_date: text('established_date'),
  evidence: text('evidence'),
  evidence_source: text('evidence_source'),
  // call_transcript / document_engagement / explicit_statement / agent_assessment
  confidence: text('confidence'),                       // HIGH / MEDIUM / LOW
  last_updated: timestamp('last_updated').defaultNow(),
  updated_by: text('updated_by'),
}, (table) => ({
  unique_lead_belief: unique().on(table.lead_id, table.belief_id),
}));

export const insertLeadBeliefsSchema = createInsertSchema(leadBeliefsTable).omit({ id: true });
export type InsertLeadBelief = z.infer<typeof insertLeadBeliefsSchema>;
export type LeadBelief = typeof leadBeliefsTable.$inferSelect;
```
