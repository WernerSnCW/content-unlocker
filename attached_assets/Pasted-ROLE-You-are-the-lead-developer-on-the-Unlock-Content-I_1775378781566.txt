```sql
CREATE TABLE output_templates (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  output_type TEXT NOT NULL,
  channel TEXT REFERENCES channels(id),
  parent_template_id TEXT REFERENCES output_templates(id),
  sections JSONB NOT NULL,
  formatting_rules JSONB NOT NULL,
  required_acus JSONB DEFAULT '[]',
  prohibited_acus JSONB DEFAULT '[]',
```

PART 4 — RULE-BASED CONTRADICTION CHECKS

Add five deterministic checks to the contradiction detector that run BEFORE the LLM pass:

```typescript
// R1: Numeric mismatch on same topic
// If two ACUs share the same topic tag and contain different numeric figures,
// flag as CRITICAL contradiction automatically

// R2: Prohibited phrase present in non-prohibited ACU
// If a prohibited ACU's content appears verbatim in a non-prohibited ACU,
// flag as CRITICAL automatically

// R3: requires_qualifier=true but qualifier absent from document
// If a fact ACU with requires_qualifier is used in a document but the
// qualifier ACU is not also present, flag as HIGH automatically

// R4: Superseded version still in use
// If acu.supersedes is populated and the superseded ACU appears in a document,
// flag as HIGH automatically

// R5: Expression variant used in wrong channel
// If a variant with variant_audience="cold" appears in a document
// generated for a warm or hot channel, flag as MEDIUM automatically
```

These run synchronously before the async Claude LLM call. If R1 or R2 fire, the LLM call can be skipped for that pair (severity is already CRITICAL).
